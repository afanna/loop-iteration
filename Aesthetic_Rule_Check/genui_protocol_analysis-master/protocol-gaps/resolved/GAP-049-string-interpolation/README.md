# GAP-049: 字符串插值方案 — 模板字符串与 format() 函数暂不支持

## 问题描述

协议中存在两种字符串插值机制：

1. **模板字符串**（反引号 `` ` `` 包裹）：`` {{ `Hello, ${$name}!` }} `` — 在 `{{ }}` 内使用反引号包裹文本，`$var` 直接嵌入或 `${expr}` 求值
2. **`format()` 函数**（已在 GAP-047 中删除）：`format("Hello, {0}", $name)` — 位置参数格式化

两种机制均增加了表达式语法的复杂度，且字符串拼接场景可通过 `+` 运算符实现（`{{ 'Hello, ' + $name }}`）。当前阶段优先简化表达式系统，暂不支持模板字符串和 format() 函数，待后续评估需求后再考虑是否恢复。

## 影响范围

- 协议章节: §4.4.2（数据类型）、§4.4.5（内置函数）、§4.5（变量系统）、§4.5.3（JSON Pointer）、§4.5.5（变量引用语法）、§4.6（EBNF 文法）、§4.7（JSON Schema）
- 测试分类: FP-04（表达式）
- 评估脚本: eval-template-interpolation.ts、eval-builtin-func.ts

## 候选方案（待评估）

### 方案A: 恢复模板字符串支持

保留模板字符串语法，用于字符串拼接场景：
```json
{"content": "{{ `Hello, ${$name}!` }}"}
```

### 方案B: 恢复 format() 函数

提供类似 printf 的格式化函数：
```json
{"content": "{{ format('Hello, {0}!', $name) }}"}
```

### 方案C: 维持现状（仅 `+` 拼接）

字符串拼接统一使用 `+` 运算符：
```json
{"content": "{{ 'Hello, ' + $name }}"}
```
