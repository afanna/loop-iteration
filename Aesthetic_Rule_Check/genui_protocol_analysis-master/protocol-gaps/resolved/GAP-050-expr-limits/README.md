# GAP-050: 表达式安全约束 — 字符串总长度 & 括号嵌套深度上限

## 问题描述

协议 §4.4 表达式系统未定义表达式字符串长度和括号嵌套深度的上限。极端情况下 LLM 或客户端可能生成/接收超长或深度嵌套的表达式，存在运行时安全隐患（栈溢出、无限递归、DoS）。

需要在协议中增加明确的约束：
- 表达式字符串总长度 ≤ **2048** 字符
- 表达式内括号嵌套最大层数 ≤ **20** 层（超过此深度视为非法表达式，求值失败返回 `""`）

> 注：嵌套深度统计包括 `()`、`[]`、模板字符串 `${}` 中的表达式递归嵌套。所有括号类符号均参与深度计数。

## 影响范围

- 协议章节: §4.4.1（基本语法-限制说明）、§3.5（表达式约束列表）
- 测试分类: FP-04（不影响现有测试用例，现有用例不会超过此上限）

## 候选修复方案

**方案**（唯一）：
1. §4.4.1 限制说明新增 2 条约束：长度 ≤ 2048、嵌套深度 ≤ 20
2. §3.5 约束列表新增第 9 条索引引用
3. `eval/prompts/protocol-summary.md` 表达式系统章节补充约束
