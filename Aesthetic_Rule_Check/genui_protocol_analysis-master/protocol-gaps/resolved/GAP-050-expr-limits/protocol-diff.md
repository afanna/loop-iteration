# GAP-050 协议修改记录

## 修改 1: §4.4.1 限制说明 — 新增长度 & 嵌套深度约束

**位置**: §4.4.1 基本语法 → 限制说明

**修改前**:
```
* ❌ "{{ inner {{ nested }} }}" - 不允许嵌套
* ❌ "{{ expr1 }} and {{ expr2 }}" - 不允许包含多个表达式
* ❌ "{{ '123' }} + {{ $__WindowSize.width }}" - 不允许多个表达式拼接
* ✅ "{{ expr1 + ' and ' + expr2 }}" - 使用字符串连接形成单个完整表达式
* ✅ "{{ '123' + $__WindowSize.width }}" - 在一个表达式内完成所有计算
```

**修改后**:
```
* ❌ "{{ inner {{ nested }} }}" - 不允许嵌套
* ❌ "{{ expr1 }} and {{ expr2 }}" - 不允许包含多个表达式
* ❌ "{{ '123' }} + {{ $__WindowSize.width }}" - 不允许多个表达式拼接
* ✅ "{{ expr1 + ' and ' + expr2 }}" - 使用字符串连接形成单个完整表达式
* ✅ "{{ '123' + $__WindowSize.width }}" - 在一个表达式内完成所有计算

**安全约束：**

| 约束 | 上限值 | 超限行为 |
|------|--------|----------|
| 表达式字符串总长度 | 2048 字符 | 求值失败，返回 `""` |
| 括号嵌套最大深度 | 20 层 | 求值失败，返回 `""` |

> 括号嵌套深度统计包括 `()`、`[]`、模板字符串 `${}` 中的所有括号类符号。深度从 0 开始计数，每进入一层括号 +1。
```

**理由**: 防止超长/深度嵌套表达式导致运行时栈溢出或 DoS。

## 修改 2: §3.5 表达式约束列表 — 新增第 9 条

**位置**: §3.5 表达式 → 约束列表末尾

**修改前**:
```
7. 表达式只能使用在值上，不能在key中使用
8. 具体可使用表达式的场景参见catalog文件
```

**修改后**:
```
7. 表达式只能使用在值上，不能在key中使用
8. 具体可使用表达式的场景参见catalog文件
9. 表达式字符串总长度不超过 2048 字符，括号嵌套深度不超过 20 层（参见 §4.4.1 安全约束）
```

**理由**: §3.5 是表达式使用层面的约束索引，应包含安全约束的引用。
