# #5 样式属性组织方式

## 1. 设计背景

- **协议章节**：3.3（样式）
- **核心问题**：样式属性是集中管理还是扁平放置？
- **评估动机**：样式组织方式影响组件结构的清晰度和 LLM 生成一致性

协议中样式属性的放置位置是核心设计决策之一。样式是否需要独立的 `styles` 对象来组织，还是直接扁平地写在组件上，抑或通过主题预设+覆盖的方式来管理？不同组织方式对 LLM 生成准确率和一致性有潜在影响。

## 2. 候选方案

| 方案 | 样式位置 | 示例 | 说明 |
|------|----------|------|------|
| **A: styles-object**（当前设计） | 独立 `styles` 字段 | `{"component": "Extended.Column", "styles": {"padding": 16, "backgroundColor": "#FFF"}}` | 当前协议设计，样式集中在 styles 对象 |
| **B: flat-props** | 扁平直接写在组件上 | `{"component": "Extended.Column", "padding": 16, "backgroundColor": "#FFF"}` | 样式和组件属性平级 |
| **C: theme-override** | 主题预设 + `styleOverrides` 覆盖 | `{"component": "Extended.Card", "theme": "card", "styleOverrides": {"padding": 16}}` | 基于 theme 的预设+局部覆盖 |

## 5. 失败分析

### 测试用例失败情况：0 失败

**所有模型、所有策略下，15 个测试用例全部通过，无任何失败。** D1-D5 维度三种方案完全一致，唯一差异出现在 D6 一致稳定性上，属于重复生成的波动而非功能失败。

### 一致性差异分析

D6 维度的微小差异反映的是模型重复生成时的等价判定波动：

| 模型 | 方案 | 语义等价率 | 说明 |
|------|------|-----------|------|
| GLM-5.1 | styles-object | 100.0% | 完全一致 |
| GLM-5.1 | flat-props | 100.0% | 完全一致 |
| GLM-5.1 | theme-override | 87.5% | 略低，theme 语义理解存在波动 |
| DeepSeek-V3 | styles-object | 87.5% | 略低，非功能性问题 |
| DeepSeek-V3 | flat-props | 87.5% | 略低，与 styles-object 相同 |
| DeepSeek-V3 | theme-override | 100.0% | 完全一致 |

**根因分析**：

1. **LLM 对样式组织方式完全不敏感** — 三种方案 MA 差异 < 1.2%，远低于可判定阈值
2. **0-shot 即 100% 准确率** — 样式组织是模型天然熟悉的概念，无需示例引导
3. **D6 差异属噪声** — theme-override 在不同模型上 D6 表现分化（GLM 偏低而 DeepSeek 偏高），不具有方向性

**优化建议**：

- 当前 styles-object 设计无需优化
- 语义清晰度是最重要的考虑因素——样式与组件逻辑属性（component、children、listeners）明确分离

## 6. 结论

| 维度 | 判定 |
|------|------|
| **推荐方案** | 维持 A (styles-object) 方案 |
| **V2 协议一致性** | 相同（Same） — 与 V2 协议设计无差异 |
| **风险** | 无 — 三方案均达 A+ 级别，MA 差异极小 |

**推荐理由**：

1. 三种方案 MA 差异极小（< 1.2%），模型对样式组织方式不敏感
2. `styles` 独立对象具有更好的语义清晰性：样式属性与组件逻辑属性（component、children、listeners）明确分离
3. 独立对象更利于后续扩展（如支持样式继承、样式预设等高级特性）
4. 与主流 UI 框架的设计模式一致（Flutter Widget.styles、React style prop、ArkUI 属性语法），LLM 训练数据覆盖充分

## 7. 如何运行

```bash
cd eval
npm install

# 完整评估（双模型）
npm run eval:style-org

# 仅运行特定模型
ONLY_MODEL=glm npm run eval:style-org
ONLY_MODEL=deepseek npm run eval:style-org
```

报告输出至 `reports/` 目录。
