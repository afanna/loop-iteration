# 什么是 GenUI

GenUI 是一个基于 OpenHarmony ArkUI 的 AI 驱动 UI 渲染框架。它让 AI 模型能够通过标准的 JSON 描述来动态生成和更新原生鸿蒙界面。

## 核心工作原理

**用户通过自然语言描述界面需求，LLM 生成 A2UI JSON，GenUI 将其渲染为原生鸿蒙界面。**

```
"帮我做一个用户信息表单"  ──►  LLM 生成 A2UI JSON  ──►  GenUI 渲染为 ArkUI 组件
```

## 核心理念

### 1. AI 生成 UI

GenUI 将 UI 描述权交给 AI 模型——LLM 根据用户意图生成符合协议的 JSON DSL，GenUI 负责解析并渲染为原生 ArkUI 组件。

### 2. A2UI 协议驱动

GenUI 遵循 **A2UI**（Agent-to-UI）开放协议。A2UI 定义了 AI 如何通过 JSON 描述 UI 组件的标准格式。GenUI 是 A2UI 协议在 OpenHarmony 平台上的实现。

### 3. 鸿蒙原生渲染

GenUI 直接使用 ArkUI 原生组件进行渲染，这意味着：
- 流畅的原生体验
- 完整的鸿蒙设计系统（深色模式、自适应布局、手势支持）

## 适用场景

| 场景 | 说明 |
|------|------|
| 智能助手对话 UI | 用户自然语言交互 → AI 动态生成卡片、表单、列表等结构化 UI |
| 动态表单生成 | 根据业务规则实时调整表单字段、校验规则和布局 |
| 数据可视化展示 | 将结构化数据（如报表、统计）动态渲染为图表、表格 |
| 多 Agent 编排界面 | 多个 AI Agent 协同工作，各自生成独立的 UI Surface |
| 个性化推荐 UI | 根据用户画像和上下文动态调整界面内容和样式 |

## 关键能力

| 能力 | 说明 |
|------|------|
| **流式渲染** | JSONL（JSON Lines）逐行传输，边接收边渲染，首屏可见延迟极低 |
| **数据绑定** | 界面与数据模型分离，数据更新自动触发 UI 刷新 |
| **双协议栈** | 同时支持 A2UI 标准协议（18 个标准组件）和鸿蒙扩展协议（21 个扩展组件 + 表达式 + 变量） |
| **多 Surface** | 支持同时管理多个 UI 区域，独立生命周期 |
| **主题支持** | 内置深色/浅色模式跟随，支持品牌色定制 |
| **可扩展** | 支持自定义组件和自定义函数，通过 Catalog 机制注册使用 |

## 目标用户

鸿蒙应用开发者：通过集成 GenUI HAR 包，为应用提供 AI 驱动的动态 UI 能力。
