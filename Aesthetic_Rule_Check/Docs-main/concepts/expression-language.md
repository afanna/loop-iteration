# 表达式语言

> **起始版本：** API Version 20
>
> 表达式语言是鸿蒙扩展协议新增能力，仅在 ohos.a2ui.extended.catalog [Catalog](catalogs.md) 下可用。

## 概述

鸿蒙扩展协议在 A2UI 标准协议之上，引入了 {{ }} 表达式语法。表达式允许在 DSL 中直接嵌入动态计算逻辑（如字符串拼接、条件判断、算术运算、函数调用等），无需预先定义中间字段或额外的函数调用。

表达式语言支持以下能力：

- **字面量**：字符串、数字、布尔值
- **运算符**：算术、比较、逻辑、三元条件（含短路求值）
- **成员访问**：点号访问、数组索引
- **变量引用**：$ 前缀变量名
- **函数调用**：函数名 + 括号传参
- **分组**：使用括号控制求值优先级

## 基础语法

表达式包裹在双花括号中，{{ }} 与内部表达式之间可以有空格：

```
{{ 表达式 }}
```

```json
{ "id": "t1", "component": "Text", "content": "{{ 'Hello' + ' World' }}" }
```

**格式要求：**

- 整个属性值必须是完整的 {{ ... }} 格式，不支持前后拼接普通文本。
- {{ }} 内部不允许出现额外的 { 或 } 字符。
- 引擎会自动去除 {{ }} 内部首尾空白。

## 支持的数据类型

| 类型 | 示例 | 说明 |
|------|------|------|
| 字符串 | {{ 'Hello' }} | 只支持**单引号** |
| 数字 | {{ 42 }}、{{ 3.14 }} | 整数和浮点数，内部统一为 double |
| 布尔值 | {{ true }}、{{ false }} | 全小写关键字 |

## 运算符

### 优先级（11 级）

优先级数值越小，结合越紧密，越先求值：

| 优先级 | 运算符 | 说明 | 示例 | 结果类型 |
|--------|--------|------|------|----------|
| 1（最高） | 字面量、标识符、( ) | 原子值 / 分组 / 函数调用 | {{ (1 + 2) * 3 }} → 9 | 由内部表达式决定 |
| 2 | . [] | 成员访问 / 数组索引 | {{ user.name }}、{{ items[0] }} | 由访问的属性/元素决定 |
| 3 | ! -（一元） | 逻辑非 / 取负 | {{ !true }} → false、{{ -5 }} → -5 | ! → 布尔值、- → 数字 |
| 4 | * / % | 乘 / 除 / 取模 | {{ 10 % 3 }} → 1 | 数字 |
| 5 | + - | 加 / 减 | {{ 1 + 2 }} → 3、{{ 'a' + 'b' }} → 'ab' | + → 数字或字符串、- → 数字 |
| 6 | < > <= >= | 大小比较 | {{ 5 > 3 }} → true | 布尔值 |
| 7 | == != | 相等 / 不等 | {{ 'a' == 'a' }} → true | 布尔值 |
| 8 | && | 逻辑与（短路） | {{ true && false }} → false | 布尔值 |
| 9 | \|\| | 逻辑或（短路） | {{ false \|\| true }} → true | 布尔值 |
| 10（最低） | ?: | 三元条件 | {{ x > 0 ? '正' : '负' }} | 由选中的分支决定 |

### 算术运算

| 运算符 | 说明 | 结果类型 | 示例（正常） | 示例（异常/边界） |
|--------|------|----------|-------------|------------------|
| + | 数值加法；任一侧为字符串时执行拼接 | 数字或字符串 | {{ 10 + 20 }} → 30（数字）<br>{{ 'x' + 1 }} → "x1"（字符串） | {{ 10 / 0 }} → 属性值无效（除零）<br>{{ 'abc' + 1 }} → "abc1"（1 转为字符串） |
| - | 减法（两侧转数字） | 数字 | {{ 20 - 8 }} → 12 | {{ 'abc' - 1 }} → -1（'abc' 转数字为 0） |
| * | 乘法（两侧转数字） | 数字 | {{ 6 * 7 }} → 42 | {{ '3' * 2 }} → 6（'3' 转数字为 3） |
| / | 除法（两侧转数字） | 数字 | {{ 100 / 4 }} → 25 | {{ 10 / 0 }} → 属性值无效（除零） |
| % | 取模（两侧转数字） | 数字 | {{ 10 % 3 }} → 1 | {{ 10 % 0 }} → 属性值无效（除零） |

**+ 的双态行为**：当任一操作数为字符串时，另一操作数自动转为字符串后拼接；否则执行数值加法。数字转字符串时去除无意义的 .0 后缀（如 3.0 → "3"），布尔值转字符串为 "true" / "false"。

| 表达式 | 求值过程 | 结果 |
|--------|----------|------|
| {{ 1 + 2 }} | 数值加法 | 3（数字） |
| {{ 'Score: ' + 95 }} | "Score: " + "95" | "Score: 95"（字符串） |
| {{ 3.0 + 'px' }} | "3" + "px" | "3px"（字符串） |
| {{ 'is: ' + true }} | "is: " + "true" | "is: true"（字符串） |
| {{ '' + 0 }} | "" + "0" | "0"（字符串） |

**除零**：/ 和 % 的右操作数为 0 时，表达式求值失败，当前属性值无效。

### 比较运算

| 运算符 | 说明 | 示例 |
|--------|------|------|
| == | 等于 | {{ 10 == 10 }} → true |
| != | 不等于 | {{ 'abc' != 'def' }} → true |
| < | 小于 | {{ 3 < 5 }} → true |
| <= | 小于等于 | {{ 5 <= 5 }} → true |
| > | 大于 | {{ 5 > 3 }} → true |
| >= | 大于等于 | {{ 10 >= 5 }} → true |

### 逻辑运算

```
&&  逻辑与（短路求值）
||  逻辑或（短路求值）
!   逻辑非
```

| 运算符 | 说明 | 示例 |
|--------|------|------|
| && | 逻辑与（短路求值） | {{ true && false }} → false |
| \|\| | 逻辑或（短路求值） | {{ false \|\| true }} → true |
| ! | 逻辑非 | {{ !true }} → false |

**短路求值**：

- A && B：当 A 为假值时直接返回 A，不再求值 B
- A || B：当 A 为真值时直接返回 A，不再求值 B

### 一元运算

| 运算符 | 说明 | 示例 |
|--------|------|------|
| - | 取负 | {{ -42 }} → -42、{{ -(-5) }} → 5 |
| ! | 取反 | {{ !false }} → true、{{ !0 }} → true |

### 三元条件

```
条件 ? 真值 : 假值
```

```json
{ "content": "{{ $__dataModel.count > 0 ? '有库存' : '已售罄' }}" }
```

三元表达式支持嵌套，实现多分支判断：

```json
{ "content": "{{ $__dataModel.score >= 90 ? '优秀' : $__dataModel.score >= 60 ? '及格' : '不及格' }}" }
```

### 分组

使用括号 () 控制求值顺序：

```json
{ "content": "{{ (1 + 2) * 3 }}" }
{ "content": "{{ ($__dataModel.price - $__dataModel.discount) * $__dataModel.quantity }}" }
```

### 成员访问

```
对象.属性
数组[索引]
```

```json
{ "content": "{{ user.name }}" }
{ "content": "{{ items[0] }}" }
```

支持链式访问：

```json
{ "content": "{{ user.address.city }}" }
{ "content": "{{ items[0].name }}" }
```

> **注意：** 成员访问和变量引用当前为 stub 实现——在 DataModel 上下文注入前，求值失败，属性值无效。详见下方[实现状态](#当前实现状态)。

## 变量引用

变量以 $ 前缀标识：

```json
{ "content": "{{ $__dataModel.user.name }}" }
{ "content": "{{ $index + 1 }}" }
```

### DataModel 变量

#### 绝对路径

```
$__dataModel.user.name
$__dataModel.products[0].price
```

#### JSON Pointer 语法

```
${/user/name}                    标准 JSON Pointer（以 `/` 开头）
${/products/0/price}             数组索引访问
```

**示例**

```json
{
  "id": "greeting",
  "component": "Text",
  "content": "{{ 'Hello, ' + $__dataModel.user.name + '！' }}",
  "fontColor": "{{ ${/color/darkColor} }}"
}
```

### 全局系统变量

| 变量 | 说明 | 可选值 |
|------|------|--------|
| $__widthBreakpoint | 当前 [窗口宽度断点](multi-device-adaptation.md#响应式断点) | xs、sm、md、lg、xl |
| $__colorMode | 当前组件深浅色状态 | light、dark |

```json
{
  "styles": {
    "fontSize": "{{ $__widthBreakpoint == 'sm' ? 12 : 16 }}",
    "fontColor": "{{ $__colorMode == 'dark' ? '#E0E0E0' : '#333333' }}"
  }
}
```

$__widthBreakpoint 的取值范围与计算规则，详见 [多设备自适应 - 响应式断点](multi-device-adaptation.md#响应式断点)。

### 循环变量

| 变量 | 说明 |
|------|------|
| $index | 当前循环索引（从 0 开始） |
| $item | 当前循环项 |

```json
{
  "id": "user_list",
  "component": "Column",
  "children": {
    "componentId": "user_item",
    "path": "/users"
  }
}
```

在 user_item 模板中：

```json
{
  "id": "user_item",
  "component": "Text",
  "content": "{{ $index + 1 }}. {{ $item.name }}"
}
```

## 内置函数

表达式支持内置函数调用。当前内置函数如下：

| 函数 | 说明 |
|------|------|
| size(array) | 计算数组长度；当参数不是数组时，返回 0 |

```json
{ "content": "{{ '总共有' + size($__dataModel.items) }}" }
```

当 items = ['手机', '耳机', '充电器'] 时，结果为 总共有3。

## 类型转换规则

### 基础类型互转

运算中需要类型对齐时，按以下规则自动转换：

| 源类型 → 目标 | 转字符串（AsString） | 转数字（AsNumber） | 转布尔值（AsBool） |
|---------------|---------------------|-------------------|-------------------|
| **字符串** | 原值 | 解析为数字，解析失败为 0 | 非空为 true，空 "" 为 false |
| **数字** | 去除无意义 .0（如 3.0 → "3"，3.14 → "3.14"） | 原值 | 非 0 且非 NaN 为 true |
| **布尔值** | true → "true"，false → "false" | true → 1，false → 0 | 原值 |

转换示例：

| 表达式 | 转换过程 | 结果 |
|--------|----------|------|
| {{ '10' - 3 }} | '10' 转数字 → 10，10 - 3 | 7（数字） |
| {{ 'hello' - 1 }} | 'hello' 转数字 → 0（解析失败），0 - 1 | -1（数字） |
| {{ 3.0 + 'px' }} | 3.0 转字符串 → "3"，"3" + "px" | "3px"（字符串） |
| {{ true + '' }} | true 转字符串 → "true"，"true" + "" | "true"（字符串） |
| {{ !'hello' }} | 'hello' 转布尔值 → true，!true | false（布尔值） |
| {{ !'' }} | '' 转布尔值 → false，!false | true（布尔值） |
| {{ !0 }} | 0 转布尔值 → false，!false | true（布尔值） |
| {{ -true }} | true 转数字 → 1，-1 | -1（数字） |

### 运算中的转换行为

| 运算符 | 行为 | 结果类型 | 示例（正常） | 示例（异常/边界） |
|--------|------|----------|-------------|------------------|
| + | 任一侧为字符串 → 另一侧转字符串后拼接；否则 → 两侧转数字后加法 | 数字或字符串 | {{ 1 + 2 }} → 3（数字）<br>{{ 'x' + 1 }} → "x1"（字符串） | {{ 'abc' + 1 }} → "abc1"（1 转为字符串） |
| - * / % | 两侧转为数字后运算 | 数字 | {{ '10' - 3 }} → 7<br>{{ 6 * 2 }} → 12 | {{ 'abc' - 1 }} → -1（'abc' 转数字为 0）<br>{{ 10 / 0 }} → 属性值无效（除零） |
| == != | 两侧转为字符串后比较 | 布尔值 | {{ 1 == 1 }} → true<br>{{ 'a' != 'b' }} → true | {{ 1 == '1' }} → true（都转为 "1"）<br>{{ 1 == true }} → false（"1" ≠ "true"） |
| < > <= >= | 两侧转为数字后比较 | 布尔值 | {{ 5 > 3 }} → true<br>{{ 3 <= 3 }} → true | {{ 'abc' > 5 }} → false（'abc' 转数字为 0） |
| && | 左侧转布尔值：为假 → 返回左侧值；为真 → 返回右侧值 | 由返回值决定 | {{ true && 'yes' }} → "yes" | {{ false && 1/0 }} → false（短路，右侧不执行） |
| \|\| | 左侧转布尔值：为真 → 返回左侧值；为假 → 返回右侧值 | 由返回值决定 | {{ false \|\| 'default' }} → "default" | {{ 0 \|\| 'fallback' }} → "fallback"（0 为假值） |
| ! | 操作数转布尔值后取反 | 布尔值 | {{ !false }} → true<br>{{ !0 }} → true | {{ !'hello' }} → false（非空字符串为真）<br>{{ !'' }} → true（空字符串为假） |
| -（一元） | 操作数转数字后取负 | 数字 | {{ -5 }} → -5<br>{{ -(-3) }} → 3 | {{ -'abc' }} → 0（'abc' 转数字为 0）<br>{{ -true }} → -1（true 转数字为 1） |
| ?: | 条件转布尔值：为真 → consequent；为假 → alternate | 由选中的分支决定 | {{ true ? 'yes' : 'no' }} → "yes" | {{ '' ? 'yes' : 'no' }} → "no"（空字符串为假）<br>{{ 0 ? 'a' : 'b' }} → "b"（0 为假） |

## 表达式使用范围

> ⚠️ **表达式仅在鸿蒙扩展协议下可用。A2UI 标准协议完全不支持表达式，{{ }} 会被当作普通字符串。**

### 前置条件

使用表达式必须**同时满足**以下两个条件：

| 条件 | 说明 | 不满足时 |
|------|------|----------|
| 使用扩展 Catalog | DSL 通过 ohos.a2ui.extended.catalog 渲染 | {{ }} 被当作普通字符串，不被识别 |
| updateComponents 消息 | 表达式仅在 updateComponents 消息中生效 | 其他消息类型中的 {{ }} 不被处理 |

### 可用的位置

在满足上述前提后，扩展组件中**标量类型的属性值**（字符串、数字、布尔值）支持使用表达式。表达式的求值结果将替代原始值。

| 位置 | 说明 | 示例 |
|------|------|------|
| 组件属性值 | 组件属性的值类型为字符串、数字、布尔值时，可用表达式 | "content": "{{ 'Score: ' + 95 }}" |
| 样式属性值 | 样式对象内部的标量属性值，可用表达式 | "fontSize": "{{ 14 + 2 }}" |

**组件属性值：**

```json
{
  "id": "title",
  "component": "Extended.Text",
  "content": "{{ 'Score: ' + 95 }}"
}
```

**样式属性值：**

```json
{
  "id": "price_tag",
  "component": "Extended.Text",
  "styles": {
    "fontSize": "{{ 14 + 2 }}",
    "fontColor": "{{ true ? '#FF0000' : '#999999' }}",
    "visibility": "{{ false ? 'visible' : 'hidden' }}"
  }
}
```

### 使用限制

| 限制项 | 说明 | 错误示例 | 实际结果 |
|--------|------|----------|----------|
| 协议限制 | 仅鸿蒙扩展协议可用，A2UI 标准协议不支持 | A2UI 标准协议 Catalog 下 "content": "{{ 'hello' }}" | content 的值为字面字符串 {{ 'hello' }}，不会被求值 |
| 消息类型限制 | 仅 updateComponents 消息中生效 | createSurface 消息中使用 {{ }} | {{ }} 作为普通字符串保留，不会被处理 |
| id 属性 | 不支持使用表达式 | "id": "{{ name }}" | id 的值为字面字符串 "{{ name }}"，不会解析为变量 |
| component 属性 | 不支持使用表达式 | "component": "{{ type }}" | component 的值为字面字符串 "{{ type }}"，无法匹配任何组件，组件渲染失败 |
| path 数据绑定值 | {"path": "..."} 中的路径值不支持表达式 | {"content": {"path": "{{ /user/name }}"}} | path 的值为字面字符串 "{{ /user/name }}"，无法匹配 DataModel 中的路径，属性值无效 |
| 事件交互行为 | 事件监听中的 call 和 as 不支持表达式 | "onClick": {"call": "{{ funcName }}"} | call 的值为字面字符串 "{{ funcName }}"，无法匹配任何函数，点击事件不触发 |
| 属性 key | 属性的键名不支持表达式，若使用则当做普通字符串 | "{{ key }}": "value" | key 为字面字符串 "{{ key }}"，组件无法识别该属性名，属性被忽略 |
| 对象类型属性值 | 当属性/样式的值类型为对象时，不支持使用表达式 | "styles": "{{ obj }}" | styles 期望对象类型，表达式结果为字符串，类型不匹配，样式不生效 |

### 正确与错误示例

✅ **正确** — 鸿蒙扩展协议下的组件属性值使用表达式：

```json
{
  "id": "user_card",
  "component": "Extended.Text",
  "content": "{{ 'Hello, ' + $__dataModel.user.name }}",
  "styles": {
    "fontSize": "{{ $__widthBreakpoint == 'sm' ? 14 : 18 }}",
    "fontColor": "{{ $__colorMode == 'dark' ? '#FFFFFF' : '#000000' }}"
  }
}
```

❌ **错误** — A2UI 标准协议下使用表达式（{{ }} 被当作普通字符串）：

```json
{
  "id": "title",
  "component": "Text",
  "content": "{{ 'Hello' }}"
}
```

> 上述 DSL 中 content 的值是字面字符串 {{ 'Hello' }}，不会被求值。

❌ **错误** — id 和 component 使用表达式：

```json
{
  "id": "{{ userId }}",
  "component": "{{ componentType }}",
  "content": "hello"
}
```

❌ **错误** — path 数据绑定中使用表达式：

```json
{
  "content": {
    "path": "{{ /user/name }}"
  }
}
```

✅ **正确**：

```json
{
  "content": {
    "path": "/user/name"
  }
}
```

❌ **错误** — 事件中的 call 使用表达式：

```json
{
  "onClick": {
    "functionCall": {
      "call": "{{ funcName }}"
    }
  }
}
```

❌ **错误** — 对象类型的属性值使用表达式：

```json
{
  "styles": "{{ $__dataModel.styleObj }}"
}
```

✅ **正确** — 对象内的各个标量属性值分别使用表达式：

```json
{
  "styles": {
    "fontSize": "{{ $__dataModel.fontSize }}",
    "fontColor": "{{ $__dataModel.fontColor }}"
  }
}
```

## 安全约束

表达式引擎设有 4 阶段安全校验，超过任一限制时求值失败，当前属性值无效：

### 输入限制

| 约束项 | 默认值 | 说明 |
|--------|--------|------|
| 表达式最大长度 | 2048 字符 | 含 {{ }} 在内的完整字符串 |
| 最大嵌套深度 | 20 层 | 括号 () 和 [] 的嵌套层数 |


## 错误处理

表达式求值失败时，当前属性值无效。引擎**不抛异常，不中断上层流程**。

### "属性值无效"的含义

当表达式求值失败时，该属性**不会获得任何有效值**，具体表现：

| 场景 | 表现 |
|------|------|
| 组件属性（如 content） | 该属性的值不会被设置，组件显示为该属性的默认行为（如空内容） |
| 样式属性（如 fontSize） | 该样式不会被应用，组件使用该属性的默认样式 |
| 组件**不会**因此崩溃或消失，仅是该属性值缺失 | — |
| 同一组件的其他正常属性不受影响 | — |
| 其他组件不受影响 | — |

### 常见的求值失败场景

**语法与运算错误：**

| 场景 | 异常输入示例 | 处理规则 | 结果 |
|------|-------------|----------|------|
| 表达式格式不正确 | "content": "hello {{ 1 + 2 }}" | 前后有普通文本，不满足完整 {{ }} 格式 | 不识别为表达式，整体作为普通字符串 |
| 包达式格式不正确 | "content": "{{ {a: 1} }}" | 内部包含 { 字符，不符合格式要求 | 属性值无效 |
| 包含非法字符或 Token | "content": "{{ 1 & 2 }}" | & 不是合法运算符（必须 &&） | 属性值无效 |
| 包含非法字符或 Token | "content": "{{ 1 = 2 }}" | = 不是合法运算符（必须 ==） | 属性值无效 |
| 包含非法字符或 Token | "content": "{{ \hello\ }}" | \` 产生非法 Token | 属性值无效 |
| 字符串未闭合 | "content": "{{ 'hello }}" | 缺少右引号，字符串未终止 | 属性值无效 |
| 括号不匹配 | "content": "{{ (1 + 2 }}" | 左括号多，括号不平衡 | 属性值无效 |
