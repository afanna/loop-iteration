# 07 场景契合度（Intent Fitness）

Version 2.0

---

# 1. 设计目标

场景契合度用于评价：

> 卡片的视觉设计是否符合当前使用场景。

注意：

本维度不是检查：

是否显示了信息。

信息完整性已经负责。

本维度回答的是：

> "最重要的信息有没有被设计成最重要。"

因此：

Intent Fitness 属于视觉评价。

不是功能评价。

---

# 6 Metric：视觉焦点契合度

## 设计目标

DSL认为：

最重要的信息。

Screenshot中：

也应该拥有：

最大的Visual Weight。

---

设：

DSL：

重要度：

```
Importance

```

Screenshot：

视觉权重：

```
VisualWeight
```

计算：

Spearman Rank Correlation。

```
ρ

=

1-

6Σdi²

/

n(n²-1)
```

其中：

di：

排序差。

---

评分：

```
Score

=

100×

(ρ+1)/2
```

范围：

0~100。

---

例如：

DSL：

```
温度

天气

城市
```

Screenshot：

```
温度

天气

城市
```

Score：

100。

---

若：

Screenshot：

```
城市

天气

温度
```

则：

Score下降。

# 7 Metric：视觉强调合理性

设计目标：

重要信息：

应该更突出。

例如：

天气：

```
26°

```

字体：

最大。

颜色：

最高对比。

面积：

最大。

属于：

优秀。

若：

空气质量

最大。

温度

最小。

则：

设计不合理。

---

定义：

计算：

Visual Weight。

排序。

计算：

Top1是否一致。

```
Top1

DSL

=

Top1

Screenshot
```

一致：

100。

否则：

按排名距离扣分。

---

# 8 Metric：信息组织合理性

设计目标：

阅读顺序：

应符合用户认知。

例如：

天气：

```
城市

↓

天气

↓

温度

↓

空气质量
```

会议：

```
开始时间

↓

结束时间

↓

地点

↓

参会人
```

计算：

Kendall Tau。

```
τ
```

评价：

阅读顺序。

---

评分：

```
Score

=

100×

(τ+1)/2
```
