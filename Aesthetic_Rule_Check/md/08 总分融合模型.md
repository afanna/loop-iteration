# 08 总分融合模型（Overall Score Model）

Version 2.0

---

# 1. 设计目标

本章定义：

HarmonyOS 卡片审美评分系统如何将多个维度融合为最终得分。

目标：

最终得分必须满足：

- 连续变化
- 可解释
- 不重复扣分
- 易于扩展
- 工程可实现

最终输出：

```

Overall Score

0~100

```

---

# 2. 一级维度

系统最终固定为五个一级维度。

| 一级维度 | 权重 | 数据来源 |
|----------|------|----------|
| 信息完整性 |20%|DSL + OCR|
| 布局美学 |25%|Screenshot|
| 视觉设计 |30%|Screenshot|
| 设计一致性 |15%|Screenshot|
| 场景契合度 |10%|Query + DSL + Screenshot|

说明：

所有维度相互独立。

禁止重复评分。

---

```

Metric

×

Weight
```

例如：

```

=

0.2Color

+

0.15Contrast

+

...

```

最终：

Dimension

↓

Overall

---

# 5. 连续评分原则

所有Metric：

统一采用：

连续评分。

禁止：

```
if

else

```

例如：

错误：

```
留白>70%

扣5

```

正确：

Gaussian。

Sigmoid。

Linear。

连续下降。

这样：

评分更加符合真实审美。

# 8. Score等级

建议：

| 分数 | 等级 | 含义 |
|------|------|------|
|95~100|S|专业设计师水平|
|90~95|A+|优秀|
|85~90|A|良好|
|80~85|B+|可发布|
|70~80|B|一般|
|60~70|C|需要优化|
|<60|D|设计较差|

说明：

等级：

仅用于：

报告展示。

不会影响计算。

---

# 9. Confidence

系统同时输出：

Confidence。

用于表示：

本次评分可信度。

例如：

OCR：

识别率：

95%。

CV：

检测率：

98%。

则：

```
Confidence

=

OCR

×

CV

```

例如：

```
0.95

×

0.98

=

0.931
```

即：

93%。

当：

Confidence：

低于：

80%。

报告：

提醒：

```
建议人工复核。
```

Confidence：

不会影响：

Overall。
