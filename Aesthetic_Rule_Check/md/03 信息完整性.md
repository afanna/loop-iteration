# 03 信息完整性（Information）

Version 2.0

---

# 1. 设计目标

信息完整性用于评价：

> DSL 中要求展示的信息是否最终正确显示在卡片上。

注意：

该维度不评价：

- 是否美观
- 字体是否漂亮
- 排版是否合理
- 是否具有视觉层级

这些均属于后续视觉评价维度。

本维度唯一职责：

**保证卡片没有漏信息。**

---

# 2. 输入数据

Information 模块输入：

```
DSL

+

OCR
```

其中：

DSL：

提供：

设计目标。

OCR：

提供：

最终显示结果。

---

# 3. 输出

输出：

InformationResult

包括：

```
Coverage

MissingText

DuplicateText

TruncatedText

Score
```

最终：

输出：

Information Score

范围：

0~100。

---

# 4. 数据提取

## 4.1 DSL 文本提取

遍历 DSL。

提取所有：

Text

RichText

Button

Label

Title

Subtitle

等组件。

仅提取：

最终需要显示给用户的文字。

例如：

```
天气

北京

25℃

今天晴

空气质量 优
```

忽略：

```
id

path

bind

color

fontSize

margin

padding
```

这些属于：

样式。

不参与检测。

---

## 4.2 OCR 文本提取

OCR 输出：

```
文字

BBox

Confidence
```

例如：

```
北京

25℃

今天晴
```

---

# 5. 文本匹配

采用：

模糊匹配。

而不是：

字符串完全一致。

例如：

DSL：

```
HarmonyOS NEXT
```

OCR：

```
HarmonyOS
```

认为：

部分匹配。

匹配方式建议：

Levenshtein Distance

或者：

RapidFuzz

Similarity

定义：

```
Similarity

=

1

-

EditDistance

/

MaxLength
```

Similarity：

≥0.9

认为：

匹配成功。

---

# 6. Metric 1：信息覆盖率

## 设计目标

评价：

DSL 中需要展示的信息，

最终有多少真正显示。

---

设：

DSL：

共有：

n

项。

成功匹配：

m

项。

定义：

```
Coverage

=

m

/

n
```

Coverage：

范围：

0~1。

---

评分：

```
CoverageScore

=

100

×

Coverage
```

例如：

```
DSL：

5

OCR：

4
```

Coverage：

```
0.8
```

Score：

```
80
```

---

# 7. Metric 2：文字截断

目标：

判断：

文本是否因为宽度不足导致显示不完整。

例如：

DSL：

```
HarmonyOS NEXT
```

OCR：

```
HarmonyOS
```

Similarity：

0.72

认为：

发生截断。

---

定义：

```
TruncationRate

=

截断数量

/

DSL文字数量
```

评分：

```
TruncationScore

=

100

×

(1-TruncationRate)
```

---

# 8. Metric 3：重复文字

目标：

评价：

是否出现重复信息。

例如：

```
天气

天气

天气
```

定义：

```
DuplicateRate

=

重复数量

/

OCR数量
```

评分：

```
DuplicateScore

=

100

×

(1-DuplicateRate)
```

---

# 9. OCR 可信度

OCR：

返回：

Confidence。

定义：

```
ConfidenceMean

=

ΣConfidence

/

N
```

若：

平均：

低于：

0.75。

报告：

提醒：

OCR识别可信度较低。

注意：

OCR 可信度：

不参与扣分。

仅用于：

报告展示。

原因：

避免：

OCR误识别造成误扣。
