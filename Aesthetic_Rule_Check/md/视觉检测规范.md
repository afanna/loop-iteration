# 附录 C 视觉检测规范（Vision Detection Specification）

Version 2.0

---

# C.1 设计目标

视觉检测模块负责：

从卡片截图中提取所有参与美学评价的视觉元素。

本模块不负责评分。

仅负责：

视觉信息提取。

输出统一的数据结构。

供所有 Metric 使用。

---

# C.2 输入

输入：

Card Screenshot

例如：

```
card.png
```

输出：

```
VisionContext
```

---

# C.3 检测对象

系统仅检测以下视觉元素。

| 类型 | 是否参与评分 |
|------|-------------|
| 文本(Text) | √ |
| 图标(Icon) | √ |
| 图片(Image) | √ |
| 按钮(Button) | √ |
| 标签(Tag) | √ |
| 卡片背景(Card Background) | √ |
| 装饰背景(Glow/Shadow) | × |
| 模糊层 | × |
| 渐变背景 | × |

原则：

所有背景装饰均不参与美学计算。

# C.4 文本检测

OCR输出：

```
文字内容

BoundingBox

Confidence

FontSize

Rotation

Color
```

例如：

```json
{
    "text":"26°C",
    "bbox":[30,45,120,88],
    "confidence":0.98,
    "font_size":34,
    "color":"#FFFFFF"
}
```

所有文本：

必须进行BBox合并。

避免：

一个标题。

识别成：

三个文本框。

# C.12 Bounding Box 合并

对于：

OCR检测。

两个BBox：

满足：

```
VerticalOverlap

>

70%

```

且：

<8px。

自动Merge。

例如：

```
Harmony

OS

```

```

```

# C.16 Detector原则

Detector：

只能：

提取。

不能：

评分。

例如：

错误：

```
留白太多

Score=80
```

正确：

```
Whitespace=0.46
```

评分：

交给：

Metric。

实现：

职责单一原则（SRP）。
