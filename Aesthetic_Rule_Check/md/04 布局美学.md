# 04 布局美学（Layout Aesthetics）

Version 2.0

---

# 1. 设计目标

布局美学用于评价：

> 卡片中的所有视觉元素是否具有合理、舒适、均衡的空间组织方式。

布局评价不关注：

- 配色
- 字体风格
- 图标设计
- 信息内容

布局仅评价：

元素之间的空间关系。

包括：

- 留白
- 边距
- 间距
- 密度
- 重叠
- 溢出

布局评价全部来源于：

Screenshot。

---

# 2. 输入

输入：

Screenshot

经过CV分析得到：

```
Card

BBox

TextBox[]

ImageBox[]

IconBox[]

ButtonBox[]

DecorationBox[]
```

统一记为：

```
Element

{

type

bbox

area

center

}
```

所有Layout计算均基于Element。

# 3. Layout Metric

Layout包含六个Metric：

| Metric | 权重 |
|---------|------|
| 留白比例 |25%|
| 边距一致性 |20%|
| 间距节奏 |20%|
| 元素密度 |15%|
| 元素重叠 |10%|
| 元素溢出 |10%|

Layout总分：

```
Layout

=

Σ

Metric

×

Weight
```

# 4 Metric：留白比例（Whitespace）

## 4.1 设计目标

优秀的HarmonyOS卡片并不是：

文字越多越好。

也不是：

越空越高级。

而是：

保持合理留白。

留白能够：

提高阅读效率。

增强视觉呼吸感。

突出重点内容。

---

## 4.2 数据提取

CV得到：

所有元素BBox。

计算：

```
Occupied Area

=

Σ

BBox Area
```

卡片面积：

```
Card Area

=

Width

×

Height
```

得到：

```
Whitespace

=

1

-

OccupiedArea

/

CardArea
```

范围：

```
0~1
```

---

## 4.3 Intent最佳值

不同Intent：

最佳留白不同。

例如：

| Intent | μ |
|----------|------|
| 天气 |0.42|
| 音乐 |0.35|
| 日历 |0.38|
| 提醒 |0.48|
| 倒计时 |0.65|

所有参数：

配置化。

---

## 4.4 数学评分

采用：

Gaussian。

```
Score

=

100

×

exp

(

-

(x-μ)^2

/

2σ²

)
```

其中：

σ：

可配置。

建议：

```
σ

=

0.12
```

这样：

偏离越远。

下降越平滑。

而不是：

突然扣分。

---

## 4.5 输出

输出：

```
Whitespace

0.46

Ideal

0.42

Score

96
```

# 5 Metric：边距一致性（Margin）

## 5.1 设计目标

边距决定：

视觉秩序。

优秀设计：

左右边距、

上下边距

应具有统一规律。

不是：

左12

右28

上15

下36

---

## 5.2 数据提取

计算：

所有元素距离：

```
Left

Right

Top

Bottom
```

得到：

```
Margin

=

[L,R,T,B]
```

---

## 5.3 数学定义

平均值：

```
μ

=

Average
```

标准差：

```
σ
```

一致性：

采用：

CV。

```
CV

=

σ

/

μ
```

CV越小：

越统一。

---

## 5.4 评分

```
Score

=

100

×

exp(-3CV)
```

例如：

CV：

```
0.05

↓

86
```

CV：

```
0.30

↓

40
```

# 7 Metric：元素密度（Density）

## 7.1 设计目标

优秀设计：

不是：

堆满。

也不是：

空荡。

---

## 7.2 定义

```
Density

=

OccupiedArea

/

CardArea
```

注意：

=

1-

Whitespace。

但是：

评分函数：

强调：

舒适度。

Whitespace：

强调：

呼吸感。

# 8 Metric：元素重叠（Overlap）

## 8.1 设计目标

文字、

图标、

按钮

不应互相遮挡。

---

## 8.2 数学定义

任意两个元素：

计算：

IoU。

```
IoU

=

Intersection

/

Union
```

得到：

最大：

IoU。

---

## 8.3 评分

不是：

IoU>0

扣8。

采用：

连续扣分。

```
Penalty

=

100

×

IoU
```

例如：

IoU：

```
0.03

↓

97
```

IoU：

```
0.25

↓

75
```

IoU：

```
0.60

↓

40
```

连续变化。
