# 09 系统架构设计（System Architecture）

Version 2.0

---

# 1. 总体架构

HarmonyOS 卡片审美评分系统采用模块化架构。

系统整体流程如下：

```

               Query
                 │
                 ▼
           Intent Parser
                 │
                 ▼
                DSL
                 │
      ┌──────────┴──────────┐
      ▼                     ▼
Information            Screenshot
 Extraction                │
                            ▼
                    CV Detection
                            │
      ┌─────────────┬──────────────┬─────────────┐
      ▼             ▼              ▼
   Layout        Visual      Consistency
      │             │              │
      └─────────────┴──────────────┘
                    │
                    ▼
             Intent Fitness
                    │
                    ▼
              Score Fusion
                    │
                    ▼
              HTML Report

```

整个流程全部自动完成。

无需人工参与。

# 3. Detector层

Detector负责：

从Screenshot中提取所有可计算数据。

输出统一的数据结构。

例如：

```
Text

BBox

Color

FontSize

```

Detector层：

不负责评分。

仅负责数据提取。

---

# 4. Metric层

Metric负责：

数学计算。

例如：

Whitespace

Spacing

Contrast

Visual Focus

等。

Metric：

输入：

Detector结果。

输出：

MetricScore。

---

# 5. Fusion层

Fusion负责：

所有Metric融合。

例如：

```
Layout

↓

91

Visual

↓

95

Intent

↓

90

↓

Overall

93
```

---

# 6. Report层

负责：

HTML。

JSON。

Markdown。

三种输出格式。

报告：

支持展开。

支持排序。

支持查看所有Metric。

---

# 7. Config层

所有参数：

全部配置化。

例如：

最佳留白。

Sigma。

Intent。

Weight。

全部来自yaml。

代码中禁止写死。

---

# 8. Result对象

所有模块统一返回：

```python
MetricResult

name

value

ideal

score

formula

details
```

所有Metric保持一致。

方便HTML自动渲染。

---

# 9. 扩展原则

新增Metric：

仅需：

新增一个Python文件。

无需修改其它模块。

例如：

```
metrics/

shadow.py
```

即可接入。

系统满足：

开放封闭原则（OCP）。
