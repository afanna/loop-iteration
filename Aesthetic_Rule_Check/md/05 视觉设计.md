# 05 视觉设计（Visual Design）

Version 2.0

---

# 1. 设计目标

视觉设计用于评价：

> 一张卡片是否具有优秀的视觉表现力。

视觉设计评价：

不是评价内容。

不是评价布局。

而是评价：

用户第一眼看到后的视觉感受。

包括：

- 色彩
- 对比
- 视觉焦点
- 视觉重心
- 图文比例
- 阅读路径
- 层级强调

所有评价均来自：

Screenshot。

---

# 2. 输入

输入：

Screenshot

CV检测得到：

```
Element

{

bbox

type

color

area

center

fontSize

fontWeight

}
```

所有Visual计算均基于Element。

# 3. Visual Metric

Visual包含七个Metric。

| Metric | 权重 |
|---------|------|
| 色彩协调性 |20%|
| 对比度 |15%|
| 视觉焦点 |20%|
| 视觉重心 |15%|
| 图文比例 |10%|
| 阅读路径 |10%|
| 层级突出度 |10%|

# 4 Metric：色彩协调性

## 4.1 设计目标

优秀卡片：

颜色不会很多。

颜色之间具有统一关系。

而不是：

颜色越丰富越高级。

---

## 4.2 数据提取

截图：

KMeans聚类。

例如：

```
K=5
```

得到：

```
Color1

Color2

Color3

...

```

转换：

LAB颜色空间。

因为：

LAB距离更符合人眼感知。

---

## 4.3 数学定义

计算：

所有颜色之间平均距离。

```
D

=

ΣΔE

/

N
```

ΔE：

采用：

CIEDE2000。

---

## 4.4 最佳范围

HarmonyOS卡片：

通常：

2~4个主色。

平均ΔE：

保持：

18~45。

颜色：

既有区别。

又保持统一。

---

## 4.5 评分

采用：

Gaussian。

```
Score

=

100

×

exp

(-(D-μ)^2

/

2σ²)
```

---

# 5 Metric：文字对比度

## 5.1 设计目标

保证：

文字容易阅读。

---

## 5.2 数据提取

OCR文字BBox。

提取：

文字颜色。

背景颜色。

转换：

Relative Luminance。

根据WCAG：

```
L

=

0.2126R

+

0.7152G

+

0.0722B
```

计算：

Contrast Ratio：

```
CR

=

(L1+0.05)

/

(L2+0.05)
```

其中：

L1>L2。

---

## 5.3 最佳值

普通文字：

```
CR>=4.5
```

标题：

```
CR>=3
```

---

## 5.4 连续评分

不是：

低于4.5扣5分。

采用：

Sigmoid。

```
Score

=

100

/

(1+e^{-k(CR-4.5)})
```

阅读体验越好。

分数越高。

---

# 6 Metric：视觉焦点

## 6.1 设计目标

用户第一眼：

应该知道：

哪里最重要。

优秀卡片：

只有一个主要视觉中心。

---

## 6.2 数据提取

每个元素：

计算：

Visual Weight。

定义：

```
Weight

=

Area

×

Contrast

×

FontScale

×Importance
```

Importance：

根据Intent：

配置。

例如：

天气：

温度：

Importance=2。

图标：

1。

副标题：

0.5。

---

## 6.3 焦点集中度

设：

最大Weight：

```
Wmax
```

全部：

```
ΣW
```

定义：

```
Focus

=

Wmax

/

ΣW
```

---

## 6.4 最佳值

建议：

```
0.35

~

0.60
```

太低：

没有重点。

太高：

压制其它信息。

---

# 7 Metric：视觉重心

## 7.1 设计目标

卡片：

视觉重量：

不能偏向一侧。

否则：

容易产生：

失衡感。

---

## 7.2 数学定义

所有元素：

计算：

Center of Mass。

```
Cx

=

ΣWiXi

/

ΣWi

Cy

=

ΣWiYi

/

ΣWi
```

得到：

视觉质心。

---

## 7.3 偏移

计算：

```
Distance

=

√

(dx²+dy²)
```

---

## 7.4 评分

Gaussian。

距离越小。

分越高。

---

# 8 Metric：图文比例

## 8.1 设计目标

图片：

不能压制文字。

文字：

也不能堆满。

---

## 8.2 定义

```
ImageArea

/

TextArea
```

根据Intent：

配置最佳值。

例如：

天气：

```
1.2
```

音乐：

```
2.5
```

会议：

```
0.6
```

Gaussian评分。

---

# 9 Metric：阅读路径

## 9.1 设计目标

用户：

阅读：

应自然。

无需寻找。

---

## 9.2 数据提取

按：

左→右

上→下

排序。

建立：

阅读序列。

```
E1

↓

E2

↓

E3

↓

...
```

计算：

路径总长度。

```
Length

=

ΣDistance
```

再计算：

方向变化。

Direction Change。

---

## 9.3 评分

路径：

越顺畅。

越高。

方向频繁折返。

下降。

---

# 10 Metric：层级突出度

## 10.1 设计目标

卡片：

必须：

存在：

视觉层级。

不能：

所有文字：

一样大。

一样亮。

一样粗。

---

## 10.2 数据提取

计算：

每个文字：

Visual Weight。

得到：

```
W1

W2

...

```

---

## 10.3 Gini系数

采用：

Gini。

评价：

层级。

```
G

=

ΣΣ|xi-xj|

/

2n²μ
```

G越高。

说明：

层级越明显。

---

## 10.4 最佳值

建议：

```
0.28

~

0.45
```

过低：

没有重点。

过高：

只有一个元素。
