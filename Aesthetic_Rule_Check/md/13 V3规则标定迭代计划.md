# HarmonyOS 卡片审美规则评分系统 V3 迭代计划

本文件用于指导新项目或新对话中的 AI 继续推进规则评分系统迭代。

当前已有 V2 项目：`Aesthetic_Rule_Check`。它是一个纯规则评分系统，通过 OCR、视觉检测、DSL 解析和规则公式，对 HarmonyOS 卡片截图进行审美评分。

V3 的目标不是重新发明评分系统，也不是把视觉大模型接入线上评分，而是建立一个离线规则标定系统，利用视觉模型评分结果反向优化 V2 规则系统，使规则评分越来越接近设计师和视觉模型的审美判断。

---

# 1. 项目定位

项目名称：

```text
HarmonyOS 卡片审美规则评分系统 V3
规则标定与迭代系统
```

系统定位：

```text
Aesthetic_Rule_Check = 线上规则评分系统
aesthetic-v4-vlm-judge-package-20260624 = 离线视觉 Teacher 评分系统
V3 Rule Calibration System = 离线规则标定系统
```

V3 不替代 V2 评分。

V3 是 V2 的离线标定器。

---

# 2. 核心原则

必须严格遵守以下原则：

1. **线上评分只使用规则引擎**

   视觉模型不能参与线上评分。

2. **视觉模型只作为离线 Teacher**

   视觉模型只用于生成参考评分、发现规则偏差、辅助人工分析。

3. **优先优化参数，不优先修改公式**

   优先调整：

   ```text
   权重
   阈值
   惩罚系数
   理想区间
   上下限
   ```

   不要轻易修改 Metric 公式。

4. **所有规则变更必须人工审核**

   系统只能生成建议，不能自动改规则。

5. **必须保留可解释性**

   每一项建议都必须说明：

   ```text
   哪些样本支持这个结论
   当前误差是多少
   影响哪些 Metric
   调整后预期收益是什么
   可能带来什么风险
   ```

6. **不要依赖报告中的可信度字段**

   当前 V2 报告中的可信度主要是 OCR / 检测可靠性，不等于评分准确率。V3 报告中可以取消展示该字段，内部可保留作为 debug 信息。

```text

visual        视觉设计
```

当前等级只用于报告展示，不参与评分。

```text
Aesthetic_Rule_Check/md/
```

# 4. V3 总体目标

建立 Rule Calibration System，用于离线分析：

```text
同一张卡片截图
    ↓
规则评分 Rule Score
    ↓
视觉教师评分 Teacher Score
    ↓
差异分析
    ↓
Metric 级归因
    ↓
权重 / 阈值 / 惩罚参数建议
    ↓
人工审核
    ↓
规则版本升级
```

V3 不负责线上评分。

V3 只负责分析、归因和生成规则优化建议。

---

# 5. 第一阶段：跑通闭环

第一阶段不要追求大规模数据，先跑通最小闭环。

目标数据量：

```text
50 - 100 张卡片
```

每个样本必须保存：

```json
{
  "sample_id": "unique_id",
  "scene_type": "weather/calendar/music/news/stock/health/etc",
  "query": "用户查询",
  "dsl_path": "DSL 文件路径",
  "screenshot_path": "截图路径",
  "rule_result_path": "规则评分 JSON",
  "teacher_result_path": "视觉模型评分 JSON",
  "rule_version": "规则版本",
  "teacher_version": "教师模型版本",
  "created_at": "时间"
}
```

第一阶段验收标准：

```text
1. 能批量读取截图和 DSL
2. 能运行规则评分
3. 能运行视觉 Teacher 评分
4. 能保存两边标准化结果
5. 能生成差异报告
6. 能定位误差最大的样本
```

---

# 6. 第二阶段：Metric 级差异分析

不能只比较 Overall Score。

必须比较：

```text
总体分
一级维度分
二级 Metric 分
Metric 原始值
Metric 权重
Metric 惩罚项
Metric 状态
```

每个样本生成结构化对比：

```json
{
  "sample_id": "xxx",
  "overall": {
    "rule": 84.2,
    "teacher": 91.0,
    "error": 6.8
  },
  "dimensions": {
    "layout": {
      "rule": 82,
      "teacher": 90,
      "error": 8
    },
    "visual": {
      "rule": 88,
      "teacher": 92,
      "error": 4
    }
  },
  "metrics": [
    {
      "metric": "whitespace_ratio",
      "rule_score": 65,
      "teacher_score": 90,
      "error": 25,
      "raw_value": 0.72,
      "weight": 8,
      "suggestion": "规则可能低估大留白设计"
    }
  ]
}
```

第二阶段验收标准：

```text
1. 能输出 MetricError.csv
2. 能输出 DimensionError.csv
3. 能输出 OverallError.csv
4. 能按误差绝对值排序
5. 能找出长期高估和长期低估的 Metric
```

# 7. 第三阶段：参数标定建议

V3 优先输出参数建议，不直接改规则。

可标定参数包括：

```text
weight          权重
threshold       阈值
penalty         惩罚系数
ideal_range     理想区间
cap/floor       上下限
```

必须区分三类建议：

```text
安全建议：数据充分，方向稳定，可以优先人工审核
观察建议：有趋势，但样本不足
风险建议：可能提升整体拟合，但会伤害某些场景
```

输出示例：

```json
{
  "metric": "whitespace_ratio",
  "current": {
    "ideal_min": 0.20,
    "ideal_max": 0.70
  },
  "suggested": {
    "ideal_min": 0.22,
    "ideal_max": 0.78
  },
  "evidence": {
    "sample_count": 1280,
    "mean_error_before": 7.4,
    "mean_error_after_estimated": 4.1,
    "affected_scenes": ["weather", "calendar", "health"]
  },
  "risk": "medium",
  "reason": "教师模型在多场景中普遍更认可大留白设计"
}
```

第三阶段验收标准：

```text
1. 能输出 ThresholdSuggestion.json
2. 能输出 WeightSuggestion.json
3. 能输出 PenaltySuggestion.json
4. 每条建议都有样本数量、误差变化和风险说明
```

# 8. 第四阶段：相关性和有效性评估

每个 Metric 都需要计算：

```text
Pearson 相关系数
Spearman 排名相关
MAE
RMSE
Bias 平均偏差
Error Std 误差标准差
```

解释规则：

```text
Pearson > 0.8   高相关
0.5 - 0.8       中等相关
0.3 - 0.5       弱相关
< 0.3           低相关
```

如果某个 Metric 在 1000+ 样本中长期低相关：

```text
Correlation < 0.2
```

则进入重点分析列表，建议：

```text
降低权重
重设阈值
合并到其他 Metric
标记为低价值 Metric
```

但不要自动删除。

# 9. 第五阶段：缺失 Metric 发现

如果出现大量样本：

```text
Rule Score 明显低于 Teacher Score
但现有 Metric 无法解释差异
```

则说明可能存在缺失审美因素。

系统需要生成：

```text
MissingMetric.md
```

候选缺失 Metric 可能包括：

```text
视觉焦点
视觉节奏
完成度
卡片呼吸感
信息主次感
组件精致度
品牌感
情绪氛围
```

每个缺失 Metric 建议必须包含：

```text
现象描述
代表样本
现有规则无法解释的原因
可能的检测方式
是否适合规则化
风险
```

# 10. 数据量路线

不要一开始追求 10000 张。

按以下路线推进：

```text
50 - 100 张
验证流程闭环

300 - 500 张
发现明显规则问题

1000 张
第一轮有效标定

3000 - 5000 张
稳定调整权重和阈值

10000+ 张
生产级长期迭代
```

场景需要均衡覆盖：

```text
天气
日历
音乐
新闻
股票
健康
提醒
快递
地图
设备控制
```

每个核心场景建议：

```text
初期：50 - 100 张
稳定期：300 - 500 张
成熟期：1000+ 张
```

# 12. CalibrationReport.md 结构

报告必须包含：

```text
1. 本轮数据概览
2. 样本数量与场景分布
3. 规则版本与教师版本
4. Overall 误差统计
5. Dimension 误差统计
6. Metric 误差排名
7. 高估 Metric 列表
8. 低估 Metric 列表
9. 低相关 Metric 列表
10. 参数调整建议
11. 权重调整建议
12. 缺失 Metric 线索
13. 风险分析
14. 人工审核清单
15. 下一轮建议
```

# 13. 重要约束

开发时必须注意：

```text
不要提交 API Key
不要把视觉模型用于线上评分
不要自动修改规则配置
不要只看总分
不要忽略场景差异
不要把 Teacher Score 当成绝对真理
不要用单轮小样本大幅调整权重
不要删除可解释性
```

---

# 14. V3 成功标准

V3 的成功不是让分数变高，而是让规则评分更接近人类审美判断。

核心指标：

```text
Overall MAE 下降
Dimension MAE 下降
Metric 偏差可解释
场景间误差更均衡
规则评分与 Teacher Score 相关性提升
人工审核认可参数建议
线上规则仍然稳定、可解释、低成本
```

最终目标：

```text
用离线视觉模型训练和校准规则系统，
让线上仍然保持纯规则、可解释、稳定、低成本，
但评分结果逐步接近设计师审美判断。
```
