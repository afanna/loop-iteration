# 06 设计一致性（Design Consistency）

Version 2.0

---

# 1. 设计目标

设计一致性用于评价：

> 卡片中的所有视觉元素是否遵循统一的设计语言（Design Language）。

一致性评价的是：

整体是否具有统一感。

而不是：

布局是否合理。

也不是：

颜色是否漂亮。

例如：

下面两张卡片：

第一张：

字体统一。

字号统一。

图标统一。

圆角统一。

即使布局一般，

仍然具有较好的设计一致性。

第二张：

字体混乱。

字号杂乱。

图标风格混搭。

圆角各不相同。

即使布局优秀，

一致性仍然较差。

因此：

Consistency 与 Layout、Visual 相互独立。

# 3. Consistency Metric

Consistency 包含五个评价指标。

| Metric | 权重 |
|---------|------|
| 字体统一性 |25%|
| 字号统一性 |20%|
| 图标风格统一性 |20%|
| 圆角统一性 |20%|
| 装饰元素一致性 |15%|

# 4 Metric：字体统一性

## 4.1 设计目标

优秀卡片通常不会混用多种字体。

例如：

```
HarmonyOS Sans

HarmonyOS Sans

HarmonyOS Sans
```

属于：

统一。

而：

```
HarmonyOS Sans

Arial

Times

```

属于：

混乱。

---

## 4.2 数据提取

OCR：

结合字体分类器。

得到：

```
Font Family

```

例如：

```
HarmonyOS Sans

HarmonyOS Sans

HarmonyOS Sans
```

---

## 4.3 数学定义

设：

共有：

N

段文字。

字体类别：

```
f1

f2

...

fk
```

定义：

字体统一率：

```
Consistency

=

最大类别数量

/

N
```

例如：

```
8

段文字

7

段

Harmony

1

段

Arial

```

得到：

```
7/8

=

0.875
```

---

## 4.4 评分

```
Score

=

100×

Consistency
```

# 5 Metric：字号统一性

## 5.1 设计目标

字号可以有层级。

但：

层级应有限。

例如：

```
12

16

24
```

很好。

而：

```
11

13

14

15

17

18

20

23

```

属于：

杂乱。

---

## 5.2 数据提取

OCR：

估计：

Font Size。

量化：

```
Round(size/2)*2
```

减少OCR误差。

---

## 5.3 数学定义

统计：

字号等级数：

```
Level Count
```

建议：

```
2~4级
```

采用Gaussian评分。

---

# 6 Metric：图标统一性

## 6.1 设计目标

优秀设计：

图标：

应来自：

同一设计语言。

例如：

全部：

Outline。

或者：

全部：

Filled。

不能：

混搭。

---

## 6.2 数据提取

CV：

提取：

所有Icon。

计算：

```
Stroke Width

Corner Radius

Aspect Ratio

```

聚类。

---

## 6.3 数学定义

计算：

Silhouette Score。

评价：

是否属于：

同一风格。

```
Score

=

Cluster Quality
```

---

# 7 Metric：圆角统一性

## 7.1 设计目标

优秀设计：

圆角：

一般：

一种。

最多：

两种。

例如：

```
8

8

8

8
```

优秀。

---

## 7.2 数据提取

CV：

拟合：

Rectangle。

估计：

Corner Radius。

---

## 7.3 数学定义

统计：

Radius种类。

Gaussian评分。

---

# 8 Metric：装饰元素一致性

## 8.1 设计目标

背景光效、

渐变、

阴影、

玻璃效果

应属于：

同一视觉风格。

例如：

全部：

柔和渐变。

全部：

低透明。

而不是：

一半拟物。

一半扁平。

---

## 8.2 数据提取

CV：

识别：

Decoration。

提取：

```
Blur

Opacity

Gradient

Shadow
```

形成：

Feature Vector。

聚类。

---

## 8.3 数学定义

采用：

Feature Variance。

```
Variance↓

Score↑
```
