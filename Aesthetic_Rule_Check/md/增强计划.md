# 第11章 V2.1 增强计划（Enhancement Plan）

Version 2.1

---

# 11.1 修改目标

V2 版本已经建立了完整的美学评分体系，包括：

- 信息完整性
- 布局美学
- 色彩美学
- 视觉层级
- 一致性
- 场景契合度

经过评审，决定进一步增强系统的美学表达能力，使评分更加符合真实设计师的审美逻辑。

本章节仅增加新的 Metric，不修改已有数学公式。

---

# 11.2 新增维度：完成度（Detail Finish）

## 设计目标

完成度用于评价：

卡片是否具有统一、精致、成熟的设计系统。

它不是布局。

也不是颜色。

而是整个界面的细节是否统一。

例如：

- 图标是否统一
- 圆角是否统一
- 间距是否统一
- 字体是否统一
- 按钮是否统一

这是设计师评价 UI 时的重要指标。

建议权重：

```
15 分
```

---

## Metric 1：圆角一致性

提取：

所有组件圆角：

```
R = {r1,r2,...,rn}
```

计算：

```
UniqueRadius = 去重(R)
```

评分：

```
UniqueRadius ≤ 2

100%

UniqueRadius = 3

90%

UniqueRadius = 4

70%

UniqueRadius >4

50%
```

---

## Metric 2：图标尺寸一致性

设：

```
IconSize={s1,s2,...sn}
```

计算：

```
CV = σ / μ
```

CV 越小越统一。

建议：

```
CV<0.15

优秀

0.15~0.30

良好

>0.30

扣分
```

---

## Metric 3：Padding 一致性

统计：

所有组件：

```
PaddingLeft

PaddingRight

PaddingTop

PaddingBottom
```

计算：

```
CVPadding
```

越小说明：

完成度越高。

---

## Metric 4：字体统一性

统计：

```
FontFamily

FontWeight

LetterSpacing

```

若：

超过：

```
2

种字体

```

开始扣分。

---

## Metric 5：组件风格统一

检测：

```
按钮

卡片

标签

```

是否：

使用一致：

```
圆角

阴影

描边

```

风格混搭：

降低完成度。

# 11.3 新增 Metric：视觉焦点（Visual Focus）

设计目标：

优秀卡片必须存在视觉中心。

不能：

所有元素都一样重要。

---

计算：

首先计算每个元素视觉权重：

```
Weight

=

Area

×

Contrast

×

FontWeight

×

Importance
```

然后：

```
FocusRatio

=

MaxWeight

/

ΣWeight
```

推荐：

```
0.25~0.45
```

评分最高。

若：

```
FocusRatio<0.18
```

说明：

没有视觉中心。

---

# 11.4 新增 Metric：阅读路径（Reading Flow）

设计目标：

评价用户阅读是否自然。

---

建立：

阅读序列：

```
Text1

↓

Text2

↓

Text3

↓

Button
```

计算：

真实阅读路径。

与：

标准阅读路径：

```
左上

↓

右下

```

进行比较。

采用：

Spearman Rank Correlation：

```
ρ

=

1-

6Σd²

/

n(n²-1)
```

ρ 越高：

阅读体验越自然。

---

# 11.5 新增 Metric：信息压力（Information Pressure）

设计目标：

评价：

卡片是否信息过载。

不是：

文字数量。

而是：

文字实际占据面积。

---

定义：

```
TextArea

=

Σ(TextBBoxArea)
```

计算：

```
Pressure

=

TextArea

/

CardArea
```

建议：

```
15%

~

35%

最佳
```

过高：

页面拥挤。

过低：

信息不足。

---

# 11.6 新增 Metric：视觉节奏（Visual Rhythm）

设计目标：

评价：

元素排列是否具有节奏感。

---

统计：

相邻元素间距：

```
D={d1,d2,d3...}
```

计算：

```
CV

=

σ

/

μ
```

CV 越低。

节奏越稳定。

---

# 11.7 新增 Metric：风格一致性（Style Consistency）

检测：

```
Icon

Button

Tag

Image

```

是否：

保持一致设计语言。

例如：

```
全部线性图标

全部圆角12

全部无阴影
```

统一：

得分提高。

混搭：

降低完成度。

---

# 11.8 DSL 的职责进一步限定

V2.1 起：

DSL 不参与任何视觉评分。

DSL 仅承担两个职责：

① 提供卡片应展示的信息。

用于：

信息完整度。

② 提供场景语义。

用于：

场景契合度。

除此之外：

DSL 不参与：

- 布局评分
- 色彩评分
- 一致性评分
- 完成度评分
- 阅读路径评分
- 视觉焦点评分

所有美学评价均来源于最终渲染截图。

---

# 11.9 非目标（Out of Scope）

评分系统明确不负责：

- DSL 合法性校验
- DSL 树结构分析
- ArkTS 编译检测
- HarmonyOS 渲染流程检测
- Widget 功能正确性
- 网络请求正确性
- 业务逻辑正确性
- 代码质量分析
- 性能测试

评分系统唯一目标：

**评价 HarmonyOS 卡片最终渲染结果的视觉美学质量。**

---

# 11.10 架构升级

V2 使用：

```
规则(R1.1、R2.2...)
```

作为主要评价方式。

V2.1 推荐逐步升级为：

```
评分维度
    ↓
Metric
    ↓
数学公式
    ↓
得分
```

例如：

```
布局美学
├── 留白比例
├── 边距一致性
├── 间距节奏
├── 对齐一致性
├── 阅读路径
├── 视觉重心
```

所有 Metric 必须满足：

- 可计算
- 可解释
- 可复现
- 有明确数学公式

避免依赖主观描述进行评分。
