# A2UI Render — Agent Guidelines

A HarmonyOS (ArkUI, stage mode) demo app that renders an A2UI v0.9 UI description
by driving the `@arkui-genius/genui` SDK (the published A2UI renderer).

## Repo layout

- `entry/` — the only Hvigor module. `apiType: stageMode`, `deviceTypes: [phone]`,
  `targetSdkVersion: 6.1.1(24)`, `compatibleSdkVersion: 6.1.0(23)`.
- `AppScope/app.json5` — bundle name `com.example.a2ui`, version `1.0.0` (code `1000000`).
- `pkg_outer/A2UIRender/` — vendored copy of the SDK **source tree** for reference only.
  The app depends on the published `@arkui-genius/genui` package, not this folder.
  `pkg_outer/A2UIRender/AGENTS.md` describes that SDK; it does **not** apply here.
- No root `package.json` / no npm. Tooling is DevEco Studio + Hvigor + ohpm.

## Real entrypoints

- App ability: `entry/src/main/ets/entryability/EntryAbility.ets` (loads `pages/Index`).
- Main page: `entry/src/main/ets/pages/Index.ets`. It:
  1. Builds a `SurfaceController` with `Catalog.extended()` (A2UI native 18-component set).
  2. Registers an `ActionReceiver` (currently just logs the action string).
  3. Reads `rawfile/sample.jsonl`, JSON-decodes the array, and feeds each entry
     via `controller.onReceive(JSON.stringify(msg))`.
  4. Renders the surface with `<UIRendererComponent surfaceControllerBound={...} />`
     and tears the controller down in `aboutToDisappear()`.
- A2UI DSL fixture: `entry/src/main/main/resources/rawfile/sample.jsonl` (A2UI v0.9
  messages — `createSurface`, `updateComponents`, `updateDataModel`). The current
  sample renders a "low-power" battery card.
- `entry/src/main/ets/pages/DemoUtils.ets` — helper that parses case files delimited
  by `===CASE_DESCRIPTION_START===` / `===CASE_JSON_START===` markers. Present but
  not wired into `Index.ets`; available for richer demo flows.
- Module manifest: `entry/src/main/module.json5`. Pages registry:
  `entry/src/main/resources/base/profile/main_pages.json` (only `pages/Index`).

## Key dependencies

Pinned in root `oh-package.json5`:
- `@arkui-genius/genui@0.7.0-alpha.4` — the renderer SDK this app demonstrates.
- Dev: `@ohos/hypium@1.0.25` (test framework), `@ohos/hamock@1.0.0`.

`entry/oh-package.json5` currently has no dependencies — UI work happens in `entry/`.

## Build / test / lint

HarmonyOS toolchain (hvigor wrapper + ohpm). Commands run from the repo root.

```bash
# Install deps (regenerates oh_modules/)
ohpm install

# Build the default HAP
hvigorw clean all
hvigorw --mode module -p product=default assembleHap --analyze=normal --parallel --incremental --daemon

# ArkTS unit tests (local, in-process)
hvigorw test -p module=entry@default -p coverage=false

# Run a single test file
hvigorw test -p module=entry@default -p coverage=false --test-file=List.test

# Lint (covers **/*.ets, including the vendored SDK tree)
hvigorw lint -p module=entry@default
```

There is no C++ build, no `npm`/`pnpm`, and no Node-based formatter. Open and
run the project in DevEco Studio for the emulator flow; `local.properties` is
auto-generated by DevEco — do not edit it by hand and do not commit it.

## Tests in this repo

Both suites are stock hypium templates (an `assertContain('abc','b')` smoke check)
and have no real coverage yet. Extend them as functionality grows:
- Local unit: `entry/src/test/LocalUnit.test.ets` (entry point: `entry/src/test/List.test.ets`).
- Instrumented: `entry/src/ohosTest/ets/test/Ability.test.ets`
  (entry point: `entry/src/ohosTest/ets/test/List.test.ets`).

## Lint config

`code-linter.json5` enables `plugin:@performance/recommended` and
`plugin:@typescript-eslint/recommended`, plus `@security/no-unsafe-*` crypto
rules (all `error`). Scope is `**/*.ets`; the vendored SDK under
`pkg_outer/A2UIRender/` is **not** in the ignore list, so changes there will
trigger lint — be aware when editing inside that folder.

## Build / strict-mode gotchas

`build-profile.json5` sets `strictMode.caseSensitiveCheck: true` and
`useNormalizedOHMUrl: true`. New files in `entry/src/main/ets/...` must keep
case-correct paths and use normalized OHM URLs.

`useNormalizedOHMUrl` means imports should resolve through the ohpm module
URL scheme (e.g. `@arkui-genius/genui`), not relative paths into `oh_modules/`.

## Common pitfalls

- Editing `pkg_outer/A2UIRender/` does **not** change the running app — only the
  published `@arkui-genius/genui` package does. Treat that folder as read-only
  reference; mirror SDK changes by bumping the `oh-package.json5` version.
- `rawfile/` assets are read via `getRawFileContentSync` and `util.TextDecoder`.
  Files must be UTF-8 and the JSONL is parsed as a single JSON array, not
  line-delimited (despite the `.jsonl` name). See `Index.ets:27-32`.
- `Index.ets` is currently the only registered page. Adding a page requires
  updating `main_pages.json` and `module.json5` if you change the entry ability.
- `entry/build-profile.json5` keeps `obfuscation.enable: false` for the
  `release` build; do not enable it without a reviewed `obfuscation-rules.txt`.
